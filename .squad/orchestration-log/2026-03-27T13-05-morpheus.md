## Key Decisions Made

1. **Arena Growth Strategy: Realloc-Copy, Not Linked Chunks**
   - Allocate 2× buffer, copy old arena, free old buffer on growth
   - Keeps all arena memory contiguous
   - Simplifies pointer arithmetic and array relocation
   - Linked-chunk arenas would avoid copies but complicate array push

2. **8-Byte Alignment for All Allocations**
   - Every `bc_arena_alloc` rounds up to 8-byte boundaries
   - Satisfies alignment for i64, f64, pointers without per-type logic

3. **Result Type via Macro (Not void*)**
   - `BC_RESULT_DECL(ok_type, err_type, name)` generates concrete tagged union
   - Compiler emits one `BC_RESULT_DECL` per distinct `Result<T, E>` instantiation
   - Type safety without casts

4. **i32 Overflow Detection via i64 Widening**
   - `bc_mul_i32` widens operands to i64 before multiplying
   - Checks if result fits in i32 range
   - Simple, correct, zero undefined behavior
   - i64 mul uses careful case-analysis (no portable 128-bit in C99)

5. **Panic Tests via fork()**
   - Test suite uses `fork()` to verify panics exit(1)
   - Windows tests skipped (no fork); Linux CI runs full suite
   - Acceptable for this phase
